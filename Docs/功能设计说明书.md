## 2 系统概述

### 2.1 系统说明

- **系统名称**：NyxGuard — 女生夜归安全守护APP
- **任务提出者**：NyxGuard 项目组
- **开发者**：NyxGuard 项目组（3-5名学生）
- **用户群**：女性用户（核心）、守护者（家人/朋友）

### 2.2 项目背景

女性夜间出行安全是普遍的社会痛点，现有方案（微信报平安、分享车牌号）存在"信息黑箱"问题。NyxGuard 通过实时位置守护、AI情感陪伴、智能预警三位一体的方案，为女生夜间出行提供全程安全保障。本项目同时作为 Technovation Girls 国际比赛参赛作品。

### 2.3 系统任务

1. 提供步行模式与乘车模式双核心守护功能，覆盖夜间出行主要场景
2. 通过AI大模型实现情境感知的关怀对话，缓解用户恐惧感
3. 提供模拟来电、警报闪光、一键SOS等应急功能，保障用户人身安全
4. 实时向守护者推送行程状态和异常事件通知

### 2.4 技术与部署说明

- Android 客户端负责地图、定位、页面交互和本地离线缓存
- 后端主线架构为 FastAPI + SQLAlchemy + JWT
- 生产数据库为 Neon Postgres，本地开发和演示可使用 SQLite fallback
- 生产部署采用 FastAPI 独立部署
- Vercel 用于承载项目官网、静态内容、文档站或演示页，不作为主业务数据库持久化层

## 3 需求说明

### 3.1 整体需求

NyxGuard 系统需满足以下整体需求：

1. **实时守护**：用户出行期间持续追踪位置并上报后端，守护者可实时查看
2. **智能预警**：自动检测超时、路线偏离等异常，主动触发预警通知
3. **AI陪伴**：通过大语言模型提供情境感知的关怀对话
4. **紧急求助**：一键SOS，自动向守护者发送位置、录音等关键信息
5. **隐蔽脱身**：模拟来电帮助用户自然脱身
6. **显性威慑**：警报闪光通过高频屏幕闪烁和震动帮助用户迅速引起周围注意
7. **离线可用**：核心定位与本地缓存在断网时仍可工作

#### 4.2.1 行程设置 NGBX001

| 项目 | 说明 |
|------|------|
| 名称 | 步行行程设置 |
| 需求编号 | NGBX001 |
| 功能描述 | 用户设置步行行程的起点、终点、预计到达时间和守护者，确认后开始步行守护 |

#### 4.2.2 实时位置追踪 NGBX003

| 项目 | 说明 |
|------|------|
| 名称 | 步行实时位置追踪 |
| 需求编号 | NGBX003 |
| 功能描述 | 行程开始后启动后台定位Service，持续追踪用户位置并上报后端，地图实时更新轨迹 |
| 参与用户 | 行程中的用户 |
| 输入 | GPS定位数据（经纬度、时间戳、精度） |
| 操作序列 | 1. 行程创建成功后自动启动前台Service，通知栏显示"NyxGuard 守护中"；2. 每10秒获取一次GPS位置；3. 地图实时更新用户蓝点和已走轨迹；4. 每30秒批量上报位置至 POST /api/trips/{id}/locations；5. 断网时缓存至Room数据库，恢复后自动补传 |
| 输出 | 地图实时显示用户位置和轨迹；后端持续接收位置数据 |
| 补充说明 | 使用Android前台Service保证后台不被系统杀死；定位使用高德定位SDK |

#### 4.2.3 超时检测 NGBX004

| 项目 | 说明 |
|------|------|
| 名称 | 步行超时检测 |
| 需求编号 | NGBX004 |
| 功能描述 | 系统根据用户设置的预计到达时间自动检测是否超时，超时后弹窗提醒并通知守护者 |
| 参与用户 | 行程中的用户、守护者 |
| 输入 | 当前时间、预计到达时间 |

| 输出 | 弹窗提醒；守护者收到超时通知（含用户最后位置） |
| 补充说明 | 超时检测在客户端本地执行，不依赖网络 |

#### 4.2.4 路线偏离检测 NGBX005

| 项目 | 说明 |
|------|------|
| 名称 | 步行路线偏离检测 |
| 需求编号 | NGBX005 |
| 功能描述 | 持续计算用户当前位置与规划路线的距离，偏离超过200米时触发预警 |
| 参与用户 | 行程中的用户、守护者 |
| 输入 | 当前GPS位置、规划路线折线坐标集 |
| 操作序列 | 1. 每次定位更新时计算当前位置到路线折线的最短距离；2. 距离超过200米判定为偏离；3. 弹出提醒"您已偏离规划路线，是否安全？"；4. 用户选择"换了条路"则更新路线基准；5. 用户选择"需要帮助"则触发SOS；6. 60秒无响应自动通知守护者 |
| 输出 | 偏离提醒弹窗；守护者收到偏离通知 |
| 补充说明 | 偏离距离计算使用点到折线段的最短距离算法；偏离检测在客户端本地执行 |

| 参与用户 | 行程中的用户、守护者 |

| 补充说明 | 到达判定半径100米，考虑GPS定位误差 |

| 操作序列 | 1. 每次定位更新时计算当前位置到路线折线的最短距离；2. 距离超过500米判定为偏离；3. 弹出醒目预警"车辆已偏离预定路线！"并触发3次短震动；4. 用户选择"司机换了条路，没事"则忽略本次偏离；5. 用户选择"我感到不安全"则触发SOS；6. 30秒无响应自动通知守护者（含位置+车辆信息） |
| 输出 | 偏离预警弹窗+震动；守护者收到偏离通知 |
| 补充说明 | 乘车偏离阈值500米，大于步行模式200米，考虑司机可能选择不同路线 |

#### 4.3.3 一键SOS NGCC005

| 项目 | 说明 |
|------|------|
| 名称 | 乘车一键SOS |
| 需求编号 | NGCC005 |
| 功能描述 | 乘车追踪页提供醒目SOS按钮，用户感到危险时一键触发紧急求助，自动录音并通知守护者 |
| 参与用户 | 行程中的用户、守护者 |
| 输入 | 用户点击SOS按钮 |
| 操作序列 | 1. 用户点击乘车追踪页底部红色SOS按钮；2. 进入5秒倒计时确认（防误触），期间可取消；3. 倒计时结束自动启动后台录音；4. 向守护者发送SOS通知（位置+车辆信息+轨迹）；5. 调用 POST /api/trips/{id}/sos 记录事件；6. 定位频率提高至每3秒一次；7. 页面显示"紧急模式已启动" |
| 输出 | 录音开始；守护者收到SOS紧急通知；后端记录SOS事件 |
| 补充说明 | SOS触发后录音静默进行，不显示明显界面；录音文件本地保存，网络可用时上传后端备份 |

## 5 界面设计

### 5.1 界面功能概览

#### 5.1.1 客户端界面

NyxGuard Android 客户端采用底部导航栏 + Fragment 架构，共4个主Tab页面，以及若干功能子页面。当前实现为自定义底部导航壳层，保留 Home / Guard / AI / Profile 四个主入口。

**底部导航栏结构**：

| Tab | 图标 | 页面名称 | 对应Fragment |
|-----|------|---------|-------------|
| 1 | 首页图标 | 首页 | HomeFragment |
| 2 | 地图图标 | 守护 | MapFragment |
| 3 | 对话图标 | AI陪伴 | ChatFragment |
| 4 | 用户图标 | 个人中心 | ProfileFragment |

**功能子页面列表**：

| 页面名称 | 入口 | 说明 |
|---------|------|------|
| 登录页 | APP启动（未登录时） | 登录/注册入口 |
| 注册页 | 登录页"注册"链接 | 新用户注册 |
| 步行设置页 | 首页"步行模式"卡片 | 设置起终点、时间、守护者 |
| 步行追踪页 | 步行设置页"开始行程" | 地图追踪+状态面板 |
| 乘车信息录入页 | 首页"乘车模式"卡片 | 录入车牌、车型、颜色 |
| 乘车追踪页 | 乘车信息录入页"开始守护" | 地图追踪+SOS按钮 |
| 模拟来电设置页 | 首页"模拟来电"卡片 | 来电人/延时/音频设置 |
| 全屏来电页 | 模拟来电倒计时结束 | 仿真来电界面 |
| 模拟通话页 | 全屏来电页"接听" | 通话计时+音频播放 |
| 警报闪光页 | 首页"警报闪光"卡片 | 全屏闪烁 + 震动 + 长按关闭 |
| 守护者管理页 | 个人中心"守护者管理" | 守护者增删查 |
| SOS倒计时页 | SOS按钮触发 | 5秒倒计时确认 |

#### 5.1.2 后台接口

FastAPI 后端不提供 Web 管理界面，仅提供 RESTful API 接口供 Android 客户端调用。API 通过 Postman/Apifox 进行调试和测试。部署上采用 FastAPI 独立部署；若存在官网、落地页或文档站，则由 Vercel 承载静态内容。

### 5.2 界面关系

#### 5.2.1 首页（HomeFragment）

**页面说明**：APP主入口页面，展示问候语、守护模式入口、快捷应急工具和主题切换入口。

**页面布局**：

- 顶部：问候语区域（根据时间段动态显示，如"晚上好，注意安全哦"）
- 中部：两张主要守护卡片 + 快捷应急工具
  - 步行模式卡片：图标 + "步行守护" + 简要描述，点击进入步行设置页
  - 乘车模式卡片：图标 + "乘车守护" + 简要描述，点击进入乘车信息录入页
  - 模拟来电卡片：图标 + "模拟来电" + 简要描述，点击进入模拟来电设置页
  - 警报闪光卡片：图标 + "警报闪光" + 简要描述，点击进入全屏闪烁与震动页
- 浮层：主题切换按钮
- 底部：红色圆角SOS按钮（悬浮，常驻显示）

**页面迁移**：

```
首页 → 步行设置页（点击步行模式卡片）
首页 → 乘车信息录入页（点击乘车模式卡片）
首页 → 模拟来电设置页（点击模拟来电卡片）
首页 → 警报闪光页（点击警报闪光卡片）
首页 → SOS倒计时页（点击SOS按钮）
```

#### 5.2.2 步行追踪页

**页面说明**：步行行程进行中的核心页面，实时展示用户位置、规划路线和行程状态。

**页面布局**：

- 顶部：状态栏（"步行守护中" + 行程计时器）
- 中部：全屏高德地图（显示规划路线折线、用户蓝点、已走轨迹、起终点标记）
- 底部：状态面板（可上滑展开）
  - 剩余距离 / 预计到达时间
  - "结束行程"按钮
  - SOS按钮（红色圆形）

**页面迁移**：

```
步行设置页 → 步行追踪页（点击"开始行程"）
步行追踪页 → 首页（到达确认后）
步行追踪页 → SOS倒计时页（点击SOS按钮）
```

#### 5.2.3 乘车追踪页

**页面说明**：乘车行程进行中的核心页面，实时展示车辆位置和行驶轨迹，提供醒目的SOS按钮。

**页面布局**：

- 顶部：状态栏（"乘车守护中" + 车牌号显示）
- 中部：全屏高德地图（显示规划路线、用户蓝点、行驶轨迹）
- 底部：车辆信息卡片（车牌/车型/颜色）+ 红色SOS大按钮

**页面迁移**：

```
乘车信息录入页 → 乘车追踪页（点击"开始乘车守护"）
乘车追踪页 → 首页（到达确认后）
乘车追踪页 → SOS倒计时页（点击SOS按钮）
```

#### 5.2.4 AI对话页（ChatFragment）

**页面说明**：AI关怀对话界面，用户可与AI助手进行文字交流，接收主动关怀消息。新版本顶部加入在线状态头部，底部输入区支持更强的沉浸式陪伴视觉。

**页面布局**：

- 顶部：标题栏（"AI陪伴"）
- 中部：聊天消息列表（气泡式布局，左侧AI消息，右侧用户消息），支持上滑加载历史
- 底部：输入栏（文本输入框 + 发送按钮）

#### 5.2.5 个人中心页（ProfileFragment）

**页面说明**：用户个人信息管理和系统设置入口，提供资料编辑、守护者管理、主题切换、语言和关于信息。

**页面布局**：

- 顶部：用户头像 + 昵称 + 手机号
- 中部：守护概览卡片 + 功能列表（纵向排列）
  - 守护者管理（进入守护者管理页）
  - 主题模式（跟随系统 / 浅色 / 深色）
  - 行程记录（查看历史行程列表）
  - 模拟来电设置（来电人/音频管理）
  - AI关怀设置（定时关怀开关/频率调整）
  - 隐私政策
  - 关于NyxGuard
- 底部："退出登录"按钮

**页面迁移**：

```
个人中心 → 守护者管理页（点击"守护者管理"）
个人中心 → 登录页（点击"退出登录"）
```
