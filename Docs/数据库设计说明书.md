## 1 文档介绍

本说明书说明 NyxGuard（女生夜归安全守护APP）的数据库设计与结构情况，用于开发人员进行项目设计，同时也为后续的数据库维护工作提供良好的使用说明，也可以作为未来版本升级时的重要参考资料。

### 1.1 文档范围

该文档主要包含系统数据库的详细设计，包括数据库系统选型、设计工具、数据库配置、命名规范、数据表结构定义、逻辑设计和物理设计。

### 1.2 读者对象

- FastAPI 后端开发人员
- Android 开发人员（Room 本地数据库部分）
- 测试人员
- 项目经理

### 1.3 术语与缩写解释

| 术语或缩写 | 解释 |

| ORM | Object-Relational Mapping，对象关系映射，通过 SQLAlchemy 实现 |

## 2 数据库环境说明

### 2.1 数据库系统

| 环境 | 数据库 | 版本 | 说明 |
|------|--------|------|------|
| 本地开发 / 演示 fallback | SQLite | 3.x | 零配置，本地文件存储，适合快速迭代与离线调试 |
| 生产环境 | Neon Postgres | PostgreSQL 14+ | 托管 PostgreSQL，支持并发访问与云环境持久化 |

通过 SQLAlchemy ORM 屏蔽两种数据库的差异，开发与生产使用相同的模型代码。

### 2.2 设计工具

| 工具 | 用途 |
|------|------|
| SQLAlchemy | Python ORM 框架，定义数据模型与关系 |
| Alembic | 数据库迁移管理 |
| DB Browser for SQLite | 开发环境数据库可视化查看 |
| pgAdmin / DBeaver | 生产环境 PostgreSQL 管理（可选） |

### 2.3 数据库配置

| 配置项 | 开发环境 | 生产环境 |
|--------|---------|---------|
| 数据库URI | sqlite:///nyxguard.db | postgresql://{user}:{pass}@{host}/{db}?sslmode=require |
| 连接池大小 | — | 5（默认） |
| 字符编码 | UTF-8 | UTF-8 |
| 时区 | UTC | UTC |

补充说明：

- SQLite 用于本地开发、演示和故障 fallback，不作为正式生产主库
- 正式生产环境使用 Neon Postgres
- 部署架构采用 FastAPI 独立部署；Vercel 可承载展示页、静态内容或文档站，但不作为业务主数据库存储层

## 3 数据库的命名规则

### 3.1 数据表名称规范

- 命名须以 `ng_` 开头（NyxGuard 缩写）
- 命名使用小写字母，单词间以下划线 `_` 分隔（snake_case）
- 命名必须清晰易懂，提取关键字
- 命名不应过长，一般不超过30个字符

示例：

| 表名 | 说明 |
|------|------|
| ng_user | 用户信息表 |
| ng_guardian | 守护者信息表 |
| ng_trip | 行程记录表 |
| ng_location | 位置数据表 |
| ng_sos_event | SOS事件表 |
| ng_chat_message | 聊天消息表 |

### 3.2 数据项名称规范

- 命名使用小写字母，单词间以下划线 `_` 分隔（snake_case）
- 命名必须清晰易懂
- 主键统一命名为 `id`
- 外键命名为 `关联表名_id`，如 `user_id`、`trip_id`
- 时间字段统一使用 `_at` 后缀，如 `created_at`、`updated_at`
- 布尔字段使用 `is_` 前缀，如 `is_active`

### 3.3 数据表结构定义

每张数据表的结构定义包含以下信息：

| 列 | 说明 |
|----|------|
| 字段名 | 数据库中的列名（snake_case） |
| 描述 | 字段的中文含义说明 |
| 数据类型 | 字段的数据类型（VARCHAR、INTEGER、FLOAT、BOOLEAN、TIMESTAMP等） |
| 是否主键 | PK 表示主键 |
| 是否可空 | NOT NULL 表示不可为空 |
| 备注 | 外键关系、默认值、约束等补充说明 |

## 4 数据库设计

### 4.1 逻辑设计

NyxGuard 系统共设计 6 张数据表，表间关系如下：

```
ng_user (1) ──< (N) ng_guardian
ng_user (1) ──< (N) ng_trip
ng_trip (1) ──< (N) ng_location
ng_trip (1) ──< (N) ng_sos_event
ng_user (1) ──< (N) ng_chat_message
```

说明：用户与守护者为一对多关系；用户与行程为一对多关系；行程与位置数据为一对多关系；行程与SOS事件为一对多关系；用户与聊天消息为一对多关系。

#### 4.1.1 用户表（ng_user）

**表结构**：

| 字段名 | 描述 | 数据类型 | 是否主键 | 是否可空 | 备注 |
|--------|------|---------|---------|---------|------|
| id | 用户ID | INTEGER | PK | NOT NULL | 自增主键 |
| nickname | 昵称 | VARCHAR(50) | | NOT NULL | 2-20个字符 |
| email | 邮箱 | VARCHAR(120) | | | 与phone二选一 |
| phone | 手机号 | VARCHAR(20) | | | 与email二选一 |
| password_hash | 密码哈希 | VARCHAR(256) | | NOT NULL | bcrypt加盐哈希 |
| avatar_url | 头像URL | VARCHAR(500) | | | 默认使用系统头像 |
| emergency_phone | 紧急联系电话 | VARCHAR(20) | | | |
| is_active | 是否激活 | BOOLEAN | | NOT NULL | 默认 true |
| created_at | 创建时间 | TIMESTAMP | | NOT NULL | 默认当前时间 |
| updated_at | 更新时间 | TIMESTAMP | | NOT NULL | 默认当前时间 |

**表关系**：无外键依赖，为其他表的基础表。

#### 4.1.2 守护者表（ng_guardian）

| phone | 守护者手机号 | VARCHAR(20) | | NOT NULL | 用于接收通知 |

#### 4.1.5 SOS事件表（ng_sos_event）

**表结构**：

| 字段名 | 描述 | 数据类型 | 是否主键 | 是否可空 | 备注 |

| content | 消息内容 | TEXT | | NOT NULL | |
| trip_id | 关联行程ID | INTEGER | | | FK → ng_trip.id，行程中对话时关联 |

### 4.2 物理设计

以下为各数据表的 CREATE TABLE 语句（PostgreSQL / Neon Postgres 语法，SQLite 兼容）。实际开发中通过 SQLAlchemy ORM 模型自动生成，此处作为参考。

#### 4.2.1 ng_user

```sql
CREATE TABLE ng_user (
    id              SERIAL PRIMARY KEY,
    nickname        VARCHAR(50)  NOT NULL,
    email           VARCHAR(120),
    phone           VARCHAR(20),
    password_hash   VARCHAR(256) NOT NULL,
    avatar_url      VARCHAR(500),
    emergency_phone VARCHAR(20),
    is_active       BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at      TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.2.2 ng_guardian

```sql
CREATE TABLE ng_guardian (
    id           SERIAL PRIMARY KEY,
    user_id      INTEGER      NOT NULL REFERENCES ng_user(id) ON DELETE CASCADE,
    nickname     VARCHAR(50)  NOT NULL,
    phone        VARCHAR(20)  NOT NULL,
    relationship VARCHAR(20)  NOT NULL,
    created_at   TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_guardian_user_id ON ng_guardian(user_id);
```

#### 4.2.3 ng_trip

```sql
CREATE TABLE ng_trip (
    id                  SERIAL PRIMARY KEY,
    user_id             INTEGER      NOT NULL REFERENCES ng_user(id) ON DELETE CASCADE,
    trip_type           VARCHAR(10)  NOT NULL,
    status              VARCHAR(20)  NOT NULL DEFAULT 'active',
    start_lat           FLOAT        NOT NULL,
    start_lng           FLOAT        NOT NULL,
    start_address       VARCHAR(200),
    end_lat             FLOAT        NOT NULL,
    end_lng             FLOAT        NOT NULL,
    end_address         VARCHAR(200),
    expected_arrive_at  TIMESTAMP,
    plate_number        VARCHAR(20),
    vehicle_type        VARCHAR(20),
    vehicle_color       VARCHAR(20),
    vehicle_note        VARCHAR(200),
    distance            FLOAT,
    started_at          TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    finished_at         TIMESTAMP
);

CREATE INDEX idx_trip_user_id ON ng_trip(user_id);
CREATE INDEX idx_trip_status ON ng_trip(status);
```

#### 4.2.4 ng_location

```sql
CREATE TABLE ng_location (
    id          SERIAL PRIMARY KEY,
    trip_id     INTEGER   NOT NULL REFERENCES ng_trip(id) ON DELETE CASCADE,
    latitude    FLOAT     NOT NULL,
    longitude   FLOAT     NOT NULL,
    accuracy    FLOAT,
    recorded_at TIMESTAMP NOT NULL,
    uploaded_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_location_trip_id ON ng_location(trip_id);
CREATE INDEX idx_location_recorded_at ON ng_location(recorded_at);
```

#### 4.2.5 ng_sos_event

```sql
CREATE TABLE ng_sos_event (
    id           SERIAL PRIMARY KEY,
    trip_id      INTEGER      NOT NULL REFERENCES ng_trip(id) ON DELETE CASCADE,
    latitude     FLOAT        NOT NULL,
    longitude    FLOAT        NOT NULL,
    address      VARCHAR(200),
    audio_url    VARCHAR(500),
    status       VARCHAR(20)  NOT NULL DEFAULT 'active',
    triggered_at TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    resolved_at  TIMESTAMP
);

CREATE INDEX idx_sos_event_trip_id ON ng_sos_event(trip_id);
```

#### 4.2.6 ng_chat_message

```sql
CREATE TABLE ng_chat_message (
    id           SERIAL PRIMARY KEY,
    user_id      INTEGER      NOT NULL REFERENCES ng_user(id) ON DELETE CASCADE,
    role         VARCHAR(10)  NOT NULL,
    content      TEXT         NOT NULL,
    trip_id      INTEGER      REFERENCES ng_trip(id) ON DELETE SET NULL,
    message_type VARCHAR(20)  NOT NULL DEFAULT 'chat',
    created_at   TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_chat_message_user_id ON ng_chat_message(user_id);
CREATE INDEX idx_chat_message_trip_id ON ng_chat_message(trip_id);
```
